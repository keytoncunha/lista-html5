Listas com HTML

  • Listas são importantes.
  • Facilitam a Leitura.
  • Organizam o conteúdo.
  • Simplificam a apresentação dos itens.

Listas Ordenadas

A HTML chama de Ordered Lists todas aquelas listas onde a ordem dos itens é algo muito importante. Um passo-a-passo para criar um bolo, uma listas de aprovados no vestibular e uma listas com carros mais caros do mundo são exemplos de listas ordenadas.

Para criar uma ordered list, vamos usar a tag <ol> para delimitar a lista e <li> (list item) para identificar cada item na lista.

Código no Editor


<ol>
        <li> Item I </li>
        <li> Item II </li>
        <li> Item III </li>
        <li> Item IV </li>
</ol>

Resultado no Navegador

  1. Item I
  2. Item II
  3. Item III
  4. Item IV

A tag <ol> possui um parâmetro type, onde configuramos o tipo de marcador da lista atual. As opções de valores para esse parâmetro são: 1 - A - a - I - i

  • 1_Valor padrão. Cria listas numeradas. ( 1,2,3,4...)
  • A_Cria listas alfabéticas em maiúscula. ( A,B,C,D...)
  • a_Cria listas alfabéticas em minúcula. ( a,b,c,d...)
  • I_Cria listas em algorismos romanos em maiúsculas. ( I,II,III,IV...)
  • i_Cria listas em algorismos romanos em minúculas. ( i,ii,iii,iv...)

Você também pode indicar o início da contagem usando o parâmetro start.

Por exemplo, a tag <ol type="I" start="5"> vai gerar itens numerados como v, vi, vii, viii, ix...

Listas não Ordenadas

Unordered Lists, também chamadas de listas com marcadores, que são aquelas onde a ordem dos itens não influenciará no significado da lista.

Para criar uma Unordered List, vamos usar a tag <ul> para delimitar a lista e a tag <li> para criar cada um de seus itens internos.

Código no Editor


<ul>
        <li> Item I </li>
        <li> Item II </li>
        <li> Item III </li>
        <li> Item IV </li>
</ul>

Resultado no Navegador

  • Item I
  • Item II
  • Item III
  • Item IV

O marcador padrão é a bolinha preta totalmente preenchida (circle), mas existe a opção de configurar a propriedade type da tag <ul>

  • disc_Padrão. Uma bola preta totalmente pintada.
  • circle_Uma bola com uma borda preta e sem preenchimento.
  • square_Um pequeno quadrado preto totalmente pintado.

Misturando as Coisas

Podemos criar listas mistas, configurando listas dentro de outras listas.

Código no Editor


<ol>
        <li> Item 1 </li>
        <li> Item 2 </li>
        <ol type="a">
            <li> Item 2.1 </li>
            <li> Item 2.2 </li>
            <li> Item 2.3 </li>
        </ol>
        <li> Item 3 </li>
        <li> Item 4 </li>
</ol>

Resultado no Navegador

  1. Item 1
  2. Item 2
    1. Item 2.1
    2. Item 2.2
    3. Item 2.3
  3. Item 3
  4. Item 4

Note que os itens da lista interna (a,b,c) está desloccado para a direita em relação ao item 2, do qual essa lista é filha.

Listas de Definições

É como se fosse um dicionário, temos os termos e as suas descrições. É uma lista sem demarcadores, mas bem útil em alguns casos.

Toda lista de definições está dentro de uma tag <dl> </dl> (definition list). Cada termo é um <dt> (definition term) e cada descrição é um <dd> (definition description). Assim como os itens da lista, essas duas últimas tags possuem fechamento opciona, segundo a referência da HTML.

Código no Editor


<dl>
    <dt>HTML</dt>
        <dd>Linguagem de Marcação de
        Hipertexto</dd>
    <dt>CSS</dt>
        <dd>Folhas de Estilos em
        Cascata</dd>
    <dt>JavaScript</dt>
        <dd>Linguagem de Programação
        Para criar interatividade<dd>
</dl>

Resultado no Navegador

HTML
Linguagem de Marcação de Hipertexto
CSS
Folhas de Estilos em Cascata
JavaScript
Linguagem de Programação Para criar interatividade

Cada navegador pode mostrar um resultado ligeiramente diferente para listas de definições, mas o que mais importa é o significado desse tipo de listagem, pois ele relaciona diretamente os pares TERMO + DESCRIÇÃO e isso nos ajuda bastante com mecanismo de busca.

IMPORTANTE! Segundo a W3C, a tag <ol> é de fechamento obrigatório. Já a tag <li> tem seu fechamento opcional a partir da HTML5.
